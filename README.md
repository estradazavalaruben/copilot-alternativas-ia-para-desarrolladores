# GitHub Copilot, Claude y Alternativas IA para Desarrolladores Spring

Curso avanzado de GitHub Copilot, Claude y herramientas IA para desarrolladores Spring que quieren acelerar desarrollo, testing, refactorización y arquitectura con IntelliJ y microservicios modernos.

---

## A quién va dirigido

Desarrolladores Java con experiencia en Spring Boot y microservicios que quieran integrar GitHub Copilot, Agent Mode, MCP y modelos IA en su flujo real de desarrollo profesional.

---

## Objetivos

- Usar GitHub Copilot con los modelos de Claude en IntelliJ (Completions, Chat y Agent Mode) para desarrollar microservicios Spring con mayor velocidad sin comprometer arquitectura, calidad ni seguridad.
- Diseñar prompts profesionales para backend que guíen planificación, validación de dominio, testing y refactorización sin delegar el criterio técnico.
- Integrar modelos IA y MCP en el flujo real de desarrollo: repositorios, PRs, tareas multiarchivo, documentación y revisión de código.
- Aplicar IA en arquitectura hexagonal, DDD, microservicios y sistemas event-driven respetando capas, coherencia de dominio y estándares de equipo.
- Establecer buenas prácticas de uso de IA en proyectos Spring que protejan seguridad, calidad del código y sostenibilidad técnica.

---

## Requisitos previos

- **GitHub Copilot**: cuenta GitHub con licencia activa de GitHub Copilot Business, Enterprise o Pro+, con Copilot Chat y selección de modelos habilitados (incluidos Claude Opus 4.5 y Claude Opus 4.6), asignada antes del curso.
- **IDE**: IntelliJ IDEA Ultimate 2026.1 con suscripción activa (la edición Community tiene soporte Spring limitado).
- **Hardware**: equipo Windows o Linux con 4 núcleos, 8 GB de RAM y 20 GB libres como mínimo.
- **Plugin**: GitHub Copilot en su última versión estable instalado en IntelliJ, con sesión iniciada en GitHub desde el IDE.
- **JDK**: JDK 21 LTS compatible (p.ej. Eclipse Temurin).
- **Herramientas**: Git configurado, GitHub Copilot CLI, Docker con virtualización por hardware habilitada en BIOS/UEFI y sistema operativo.
- **Comunicación**: Zoom Workplace desktop app actualizada y probada para seguir las sesiones en directo.
- **Red**: salida HTTPS corporativa habilitada para las URLs de GitHub Copilot (`business.githubcopilot.com` y/o `enterprise.githubcopilot.com`).

Todos los componentes (IntelliJ, Copilot, JDK, Git, Docker y Zoom) deben estar instalados y probados antes del inicio del curso.
